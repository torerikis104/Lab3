Kontrollspørsmål Lab3:

0) Fordelene med å bruke et bibliotek, er at man kan lage en hjemmeside med JavaScript
uten at man egentlig kan kode i dette prog. språket. Man trenger bare å forstå hvordan 
man implementerer noe som allerede er laget.
Man kan og spare masse tid, ettersom man kan "klippe og lime" ting man finner som man liker
og som virker for det man selv lager.

1) HTML/CSS gir mer informasjon til nettleser, slik at funksjoner som keyboard interaktivitet og liknende støttes.
Hyperlinker støttes i HTML/CSS, menikke i canvas. 
Med canvas vises innhold på samme måte uansett plattform, noe som kan skape problemer.
Skjermleser verktøy baserer seg ofte på HTML strukturen og vil derfor fungere bedre med HTML/CSS.
HTML/CSS basert kode virker bedre med søker motorer, da searchbots og spiders parser html-koden og får ut mer meningsfylt
informasjon enn om applikasjonen var basert kun på canvas.

Canvas er mer grafisk enn HTML/CSS

Canvas oppdaterer seg raskere enn HTML/CSS siden det ikke trenger å oppdatere hele den hierarkiske strukturen til HTML.

Canvas vil vise innhold likt på tvers av plattformer = plattformkonsistens for applikasjonen.

Man vil kunne ha store fordeler av å feks bruke en canvas som et bakgrunnselement, hvis dette har mye informasjon eller
animasjon. På denne måten vil siden bli kjappere å bruke, siden man ikke må laste inn alt gjennom HTML, hver gang man oppdaterer.
